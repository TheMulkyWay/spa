Welcome to my open source Angular and ASP.NET single-page application, built with:

  • ASP.NET Core and C# for cross-platform server-side code
  • Angular and TypeScript for client-side code
  • Webpack for building and bundling client-side resources
  • Bootstrap for layout and styling
  • Visual Studio tooling for layout and styling

Full Stacks demo features here:

  • Angular Responsive. Angular doesn't care what's behind the backend Web API it talks to. Here you can see both working along with all the source code. This is not a another tutorial.
    Quite soon I hope to show here the daily parts played Azure, Github and VSTS along with how developer branching, database migrations, testing all come together to help keep things on track .
  • Server-side prerendering. For faster initial loading and improved SEO, this Angular app is prerendered on the server. The resulting HTML is then transferred to the browser w here a client-side copy of the app takes over.
  • Webpack dev middleware. In development mode, there's no need to run the webpack build tool. Your client-side resources are dynamically built on demand. Updates are available as soon as you modify any file.
  • Hot module replacement. In development mode, you don't even need to reload the page after making most changes. Within seconds of saving changes to files, your Angular app will be rebuilt and a new instance injected into the page.
  • Efficient production builds. In production mode, development-time features are disabled, and the webpack build tool produces minified static CSS and JavaScript files.
  • Native and 3rd party API's. A mixture of consumming 3rd party API's such as Github, and consumming Web API's I created on .NET Core Servers and on Express.
  • Unit Testing API's and UI. A mixture of 5 different 3rd party Unit testing frameworks are used covering fakes, mocks, runners client and server side.
